# Static Public Artifact Contract

## Purpose

The public autograph catalog is generated inside the OCI runtime boundary and
served as static output. GitHub Actions builds and deploys code artifacts only.
It never reads Oracle catalog content, private Object Storage metadata, or
private media, and it never generates catalog content.

The initial public contract uses `schemaVersion: 1`. Public artifacts are
display and filtering DTOs, not serialized database rows.

## Selected Shape

Phase 5 uses split collection data plus per-item detail JSON:

| Artifact | Path | Purpose |
|----------|------|---------|
| Collection page | `/collection/index.html` | Static gallery shell |
| Collection data | `/collection.json` | Gallery cards and primary image variants |
| Facets | `/facets.json` | Signer, category, and tag filtering values |
| Item page | `/items/{item-slug}/index.html` | Static item detail shell |
| Item detail data | `/items/{item-slug}.json` | Full public-safe item detail DTO |
| Publish manifest | `/manifest.json` | Release ID, generation timestamp, artifact paths, and byte sizes |
| Generated media | `/media/{item-slug}/{image-slug}-{variant}.{ext}` | Public thumbnail and detail derivatives |

The 500-item fixture profiles three shapes:

| Shape | Bytes | Notes |
|-------|------:|-------|
| Single detail JSON | 677,361 | Smallest transfer, but requires full detail regeneration for routine changes |
| Split collection plus per-item details | 946,749 | Selected: supports explicit incremental item publishing |
| Hybrid collection plus detail bundle | 947,250 | Similar size without the per-item update advantage |

The selected split is not the smallest byte count. It is the simplest shape
that preserves browse/filter behavior while allowing an item update to replace
one detail artifact and the affected indexes.

## Versioning

`collection.json`, `facets.json`, each item detail JSON artifact, and
`manifest.json` contain `schemaVersion: 1`.

Contract changes that break an existing static reader require a schema version
increment and a full rebuild. Additive fields may remain within a version when
old readers can safely ignore them.

## Public DTO Rules

Gallery DTOs contain:

- Public item slug, title, signer, description, category, and curated tags.
- An optional primary image with public derivative variants.

Detail DTOs add:

- Public derivative images.
- Display-oriented detail groups.

Facet DTOs contain:

- Signer, category, and tag options for filtering.

Manifest entries contain:

- Public artifact path.
- Generated byte size.

## Media Paths

Public derivatives use sanitized deterministic paths:

```text
/media/{item-slug}/{image-slug}-thumbnail.webp
/media/{item-slug}/{image-slug}-detail.webp
```

The path is independent from the private original filename, private Object
Storage key, and private image UUID. Thumbnail and detail derivatives are
generated output and may be rebuilt from durable private originals.

## Privacy Deny-List

Public JSON, HTML, manifests, and media paths must not contain:

- `storageNamespace`
- `bucketName`
- `objectKey`
- Object Storage URLs or the string `objectstorage`
- OCI configuration keys such as `OCI_`
- Oracle connection details
- Private image UUIDs
- Original filenames

The Rust static contract tests generate a 500-item, multi-image fixture and scan
the rendered public artifact set for these values.

## Publishing Boundary

Catalog generation runs in the private controller inside the OCI/runtime
boundary. A release is generated as a complete candidate, validated for
artifact presence and privacy, and promoted only after validation passes.

GitHub Actions may build and deploy controller code or container images. It
must not connect to Oracle, inspect private Object Storage originals, receive
private bucket details, or publish catalog content.
