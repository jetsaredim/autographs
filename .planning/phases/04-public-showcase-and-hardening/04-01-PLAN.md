<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser-to-public-app | Anonymous users reach public pages, catalog APIs, image routes, and health routes. |
| public-edge-to-operator-api | Caddy must block operator mutation routes from public ingress. |
| app-to-private-media | App streams private Object Storage bytes without exposing storage identifiers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | public catalog/image routes | mitigate | Add headers and preserve public DTO tests proving no Object Storage URLs, keys, bucket names, namespaces, or credentials leak per D-04-19. |
| T-04-02 | Information Disclosure | /health/data | mitigate | Reduce anonymous output or require operator token for data-readiness detail in production; document chosen behavior. |
| T-04-03 | Elevation of Privilege | /api/operator/* | mitigate | Preserve app bearer-token checks and Caddy 404 block; add static regression coverage for the Caddy rule per D-04-20. |
| T-04-04 | Tampering | public headers/config | mitigate | Add deterministic tests or build checks so header changes are verified by app build and route-level assertions. |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan. |
</threat_model>
