<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Oracle catalog data -> public Client Components | Server-side records include private storage identifiers that must be stripped before public rendering. |
| Approved quote source -> anonymous public UI | Quote content becomes public page copy and must remain short, attributed, and approved. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | I | `public-view-models.ts` | mitigate | Export only app-level `id`, metadata, and `/api/catalog/{itemId}/images/{imageId}` image `src`; tests assert storage fields are absent. |
| T-03-01-02 | I | Facet derivation | mitigate | Derive only `signer`, `category`, and curated tags from published item data; do not include storage metadata or raw image fields. |
| T-03-01-03 | T | Quote inventory | mitigate | Human approval checkpoint records exact quote texts before durable storage; tests require non-empty attribution. |
| T-03-01-04 | R | Quote approval | mitigate | Execution summary records approved quote inventory so later changes are auditable in planning history. |
| T-03-01-05 | I | Direct Object Storage URL/key leakage | mitigate | Deny storage field names in public view model outputs and keep image URLs app-mediated only. |
| T-03-01-06 | I | Hidden admin placeholder exposure | accept | This plan does not create admin routes; later plan must keep placeholder non-mutating and navigation-hidden. |
| T-03-01-07 | I | Operator-only production data entry docs | accept | This plan does not touch operator docs; final docs plan must keep endpoint access tunneled and token-guarded. |
</threat_model>
