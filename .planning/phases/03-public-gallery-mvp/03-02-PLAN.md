<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Anonymous browser -> landing page | Public user can interact with `View Collection`, `Surprise Me`, and the hidden unlock. |
| Browser -> admin placeholder route | `/admin` is public if guessed but must not expose privileged behavior. |
| Server catalog data -> landing hero image | Landing visual must use app-mediated image routes only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | I | Landing image | mitigate | Render only `/api/catalog/{itemId}/images/{imageId}` image sources and never storage URLs or object keys. |
| T-03-02-02 | E | `/admin` placeholder | mitigate | Static page only; source gate forbids login, bearer token, upload, edit, publish, and mutation code. |
| T-03-02-03 | I | Visible admin nav | mitigate | No visible `Admin` nav; keyboard unlock reveals placeholder link only after `g a l l e r y` sequence. |
| T-03-02-04 | I | Anonymous public browsing | mitigate | Root page calls published-only catalog service list semantics; no include-unpublished option. |
| T-03-02-05 | I | Anti-casual image extraction | mitigate | CSS/classes prevent default context menu where practical and avoid standalone image links. |
| T-03-02-06 | I | Operator-only production data entry docs | accept | This plan does not document operator endpoints; final docs plan covers tunnel-only access. |
