<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Anonymous query string -> catalog list filters | Public users control `signer`, `category`, and `tag` query params. |
| Server catalog records -> public grid cards | Published records are transformed before rendering. |
| Browser image display -> app-mediated image route | Public images are visible but should resist casual extraction and avoid storage leakage. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | T | `/collection` search params | mitigate | Pass only `signer`, `category`, and `tag` strings into existing repository filters; no raw SQL construction in UI. |
| T-03-03-02 | I | Gallery cards | mitigate | Consume `PublicGalleryItem` only; source gate forbids storage field names in components. |
| T-03-03-03 | I | Direct Object Storage URL/key leakage | mitigate | Render image `src` from `buildPublicImageSrc` output only. |
| T-03-03-04 | I | Anonymous public browsing | mitigate | Use default `service.list()` published-only semantics and never set `includeUnpublished`. |
| T-03-03-05 | I | Anti-casual image extraction | mitigate | Add context-menu prevention on image display regions where practical, disable dragging, and avoid standalone image links. |
| T-03-03-06 | I | Collection-page Surprise Me | mitigate | Source assertion forbids `Surprise Me` on `/collection`. |
</threat_model>
