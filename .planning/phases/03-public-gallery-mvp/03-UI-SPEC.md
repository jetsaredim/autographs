## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tag text offsets, focused thumbnail outline offsets |
| sm | 8px | Compact element spacing, tag gaps, thumbnail gaps |
| md | 16px | Default element spacing, card padding, filter row gaps |
| lg | 24px | Section padding, gallery card media-to-copy gaps |
| xl | 32px | Layout gaps, detail metadata section gaps |
| 2xl | 48px | Major section breaks, landing overview bands |
| 3xl | 64px | Page-level spacing on desktop |

Exceptions: image thumbnails use stable fixed boxes of 72px mobile and 88px desktop; icon-only or compact interactive targets must be at least 44px by 44px; gallery card border radius must be 8px or less.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.35 |
| Heading | 20px | 600 | 1.2 |
| Display | 32px | 600 | 1.1 |

Rules:
- Use only these four text sizes in Phase 3 UI surfaces.
- Do not scale font size with viewport width.
- Letter spacing must be `0`.
- Display text is reserved for the landing page title and item detail title. Gallery cards, filter controls, metadata sections, and empty states use Body, Label, and Heading only.
- Long signer names, titles, category labels, tags, and certification IDs must wrap without overflowing their container.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f8f7f4` | Page background and unframed section background |
| Secondary (30%) | `#ffffff` | Gallery cards, filter panels, metadata groups, quote blocks, public header |
| Accent (10%) | `#1f6f68` | Primary CTA, selected filter chips, active thumbnail outline, focused links, small count badges |
| Destructive | `#b42318` | Destructive actions only; no destructive public actions exist in Phase 3 |

Accent reserved for: `View Collection` CTA, `Surprise Me` CTA, selected filter chips, active thumbnail state, keyboard focus rings, detail-page primary facts, and recovery actions in empty/error states.

Rules:
- Do not use decorative gradients, gradient orbs, bokeh blobs, or one-note purple, slate, beige, brown, or orange-dominant palettes.
- Use `#1f2937` for primary text, `#4b5563` for secondary text, `#d7d2c8` for borders, and `#f1efea` for muted inline surfaces.
- Maintain WCAG AA contrast for body copy and controls.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | View Collection |
| Secondary CTA | Surprise Me |
| Empty state heading | selected from approved quote list |
| Empty state body | No published autographs match this view yet. Clear the filters or return to the full collection. |
| Error state | The collection could not be loaded. Refresh the page or return to the collection. |
| Destructive confirmation | none; Phase 3 public gallery has no destructive actions |

Quote rules:
- Render movie quotes as quote blocks with quote text and attribution as separate elements.
- Use short quotes only. Previously discussed examples such as "These aren't the droids you're looking for." and "X never, ever marks the spot." are examples, not the final approved inventory.
- Phase 3 development must include researching similar short movie quotes about not finding things, presenting candidates for user review/approval, and storing approved quotes in the database or another durable approved list.
- Random public empty/not-found/error states must draw from the same approved quote source/list.
- Pair every quote state with practical recovery actions.
- Do not use long quoted passages.

State-specific recovery actions:

| State | Quote Source | Action |
|-------|--------------|--------|
| No published items | Random approved quote | View Collection disabled or hidden; Surprise Me hidden |
| Filter no results | Random approved quote | Clear filters |
| Missing or unpublished detail page | Random approved quote | Back to collection |
| Media fetch failure | Random approved quote | Back to item details or Back to collection |
| Catalog fallback/error | Random approved quote | Refresh and View collection |

## Phase 3 Surface Contract

### Landing Page

- Route: `/`.
- Brand/title: `Jared Greenwald's Autograph Gallery`.
- Purpose: a basic overview of the collection, not the full grid.
- First viewport must show the public brand, a real autograph/image-backed visual signal, and both actions: `View Collection` and `Surprise Me`.
- The landing page must reveal a hint of the next section on mobile and desktop viewports.
- `View Collection` navigates to `/collection`.
- `Surprise Me` appears only on `/` and opens one random published item detail page from the full published collection.
- Do not leave the proof-of-life "Autographs" page or delivery-spine explanation as the public landing experience.
- Public pages should include a very small footer. The footer contains an `About` link to the current architecture page; architecture-page content can be expanded later.
- The public UI must include a non-obvious admin unlock affordance in Phase 3. Do not expose a visible `Admin` navigation link.
- The unlock affordance leads only to a placeholder admin page. The placeholder communicates that collection management is coming later and must not include login, mutation, upload, edit, publish, or operator workflow behavior.
- Acceptable unlock patterns include a nearly hidden footer/brand affordance or a keyboard combination, provided it is operator-discoverable and does not invite casual visitor clicks.

### Collection Page

- Route: `/collection`.
- Render an image-forward grid of published items only.
- Grid density: 1 column under 560px, 2 columns from 560px, 3 columns from 900px, and 4 columns from 1200px when space allows.
- Each card uses one primary image, title, signer, category/IP, and up to 3 meaningful tags.
- Cards must be clickable links to `/collection/{itemId}`.
- Card image boxes use a stable aspect ratio of `4 / 5` and must not shift layout while images load.
- Card radius is 8px; no nested cards.

### Filtering

- Filtering must use URL query state so selected filters survive refresh and can be shared.
- MVP query parameters: `signer`, `category`, and `tag`, matching the existing public catalog list shape.
- Controls: a compact filter dropdown or popover containing curated public facets, plus a visible selected-filter chip row.
- Curated facets start with signer, card game/IP/category, and meaningful public tags.
- Do not expose every raw database tag or storage/object metadata value.
- Selecting or clearing filters updates the grid and result count.
- The collection page must not include `Surprise Me`.

### Item Detail Page

- Route: `/collection/{itemId}`.
- Missing or unpublished records render a warm not-found state.
- The image/image gallery is the main focus and is centered on initial detail-page view.
- Clicking the focused image reduces it slightly and reveals grouped information to the right of the image where viewport space allows; this can collapse below the image on narrow viewports.
- Essential facts appear in the revealed information area: signer, item/card title, category/IP, estimated year, and certification when present.
- Supporting metadata is grouped into these sections in this order: `Provenance`, `Certification`, `Inscription`, `Tags`, `Collection Notes`.
- Hide empty metadata fields rather than rendering blank rows.
- Use definition-list style metadata rows with labels at 14px/600 and values at 16px/400.

### Multi-Image Viewer

- Show one focused primary image and all published images as thumbnails below it.
- Clicking a thumbnail swaps it into the focused image area.
- The selected thumbnail must have an accent outline and an accessible selected state.
- Thumbnail selection must not change the URL, hash, or query string.
- No dedicated lightbox route, standalone image route, or public image browsing route is included in Phase 3.

### Anti-Casual Image Extraction

- All image `src` values must use `/api/catalog/{itemId}/images/{imageId}`.
- Public HTML, JavaScript, and rendered attributes must not expose Object Storage URLs, namespaces, bucket names, object keys, storage credentials, signed URLs, or browser-visible storage identifiers.
- Do not render visible standalone image links.
- Prevent the default context menu on public image display regions.
- Do not promise DRM. The contract is anti-casual extraction and no direct storage exposure.

### Responsive and Accessibility Contract

- Use semantic landmarks: header, main, nav, section, article, figure, and dl where appropriate.
- All interactive controls must be keyboard reachable.
- Focus styles use the accent color with at least a 2px outline.
- Images must have meaningful alt text from stored image alt text when present, otherwise a generated item-aware fallback such as `{title} signed by {signer}`.
- Filter controls must expose current state to assistive technology.
- Empty states and errors must be reachable as normal page content, not only toast messages.
