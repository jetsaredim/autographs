<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Anonymous path param -> catalog detail lookup | Public users control `{id}` in `/collection/{id}`. |
| Server catalog record -> Client image viewer | Sanitized image and metadata props cross into browser state. |
| Browser image display -> app-mediated media route | Images are visible in the browser but storage details remain server-side. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04-01 | I | Detail route | mitigate | Use `getById(id)` default published-only semantics; missing/draft records call `notFound()`. |
| T-03-04-02 | I | `ImageViewer` props | mitigate | Accept only `PublicImage[]` from sanitized view model; source gate forbids storage fields. |
| T-03-04-03 | I | Direct Object Storage URL/key leakage | mitigate | Image `src` values are app-mediated route strings only. |
| T-03-04-04 | I | Anti-casual image extraction | mitigate | Prevent context menu and dragging on focused images and thumbnails; no standalone image links or lightbox route. |
| T-03-04-05 | T | Thumbnail interaction | mitigate | Local state only; no URL/hash/query mutation. |
| T-03-04-06 | I | Quote states | mitigate | Use approved durable quote list; no unapproved hard-coded quote inventory in page files. |
| T-03-04-07 | I | Hidden admin placeholder exposure | accept | This plan does not create admin features; Plan 02 mitigation remains in force. |
</threat_model>
