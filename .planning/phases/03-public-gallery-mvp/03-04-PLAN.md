<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Dynamic route param to catalog read | Anonymous users control `{itemId}` and may request drafts or nonexistent IDs. |
| Public detail page to Client image viewer | Client Component must receive only public image references, not server storage metadata. |
| Browser media request to app-mediated image route | Image bytes are app-mediated; public UI must not create direct object links. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04-01 | Information Disclosure | `/collection/[id]` | mitigate | Use default `getById(id)` and `notFound()`; never set `includeUnpublished`. |
| T-03-04-02 | Information Disclosure | `ImageViewer` props | mitigate | Pass only public image references from view-model helpers and grep for storage fields. |
| T-03-04-03 | Tampering | metadata/quote rendering | mitigate | Render metadata and quotes as React text; avoid HTML injection. |
| T-03-04-04 | Information Disclosure | image links | mitigate | Render inline images only, prevent context menu, and forbid `href` links to image routes. |
</threat_model>
