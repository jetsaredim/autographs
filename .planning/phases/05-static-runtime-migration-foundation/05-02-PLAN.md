<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser/admin shell to Rust controller | Internet-reachable login may exist, but privileged APIs require same-origin authenticated session. |
| local operator to Rust controller | SSH tunnel or localhost calls can use CLI-friendly token/session path. |
| controller config to health responses | Health endpoints must not leak secrets or detailed Oracle/Object Storage config. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Spoofing | admin login | mitigate | Implement one admin credential source, password/token verification, secure HTTP-only cookie, and protected-route tests. |
| T-05-05 | Elevation of Privilege | `/admin/api/*` | mitigate | Require authenticated session or operator token for all non-health admin API routes. |
| T-05-06 | Information Disclosure | health/errors | mitigate | Redact env values, Oracle strings, Object Storage names, and secrets from health and error responses. |
| T-05-07 | Denial of Service | login endpoint | mitigate | Add basic in-process failed-login throttling or lockout suitable for one-admin personal use. |
</threat_model>
