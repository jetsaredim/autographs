<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| private originals to derivatives | Derivatives become public artifacts and must not carry private metadata or names. |
| candidate release to current release | Anonymous users must never see half-written releases. |
| publish logs to admin UI | Failure detail must help the operator without leaking secrets. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-12 | Information Disclosure | derivative files | mitigate | Resize/re-encode outputs, strip metadata, and deny original filenames/private IDs in paths/manifests. |
| T-05-13 | Tampering | candidate release | mitigate | Validate manifest consistency, page existence, JSON parseability, derivative existence, and privacy scan before promotion. |
| T-05-14 | Denial of Service | publish rebuild | mitigate | Keep Phase 5 derivative set to thumbnail/detail and expose full rebuild as explicit operator action. |
| T-05-15 | Repudiation | publish attempts | mitigate | Record publish status, release ID, artifact counts, bytes, started/finished timestamps, and redacted failure summary. |
</threat_model>
