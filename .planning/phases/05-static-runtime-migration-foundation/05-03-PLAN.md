<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin API to Oracle | Authenticated mutation writes collection source-of-truth rows. |
| admin API to private originals | Authenticated upload writes private Object Storage/local media. |
| private original metadata to public publish path | Original keys/filenames must not become public media names. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-08 | Tampering | seed/update API | mitigate | Validate required fields, allowed publication statuses, primary image order, and item/image ownership before persistence. |
| T-05-09 | Information Disclosure | object key generation | mitigate | Generate new private original keys as UUID-only paths and store original filename only as private metadata if needed. |
| T-05-10 | Denial of Service | upload endpoint | mitigate | Enforce content-type allow-list and byte-size limits before writing media. |
| T-05-11 | Repudiation | publish seed actions | accept | Full edit history is Phase 6; Phase 5 records minimal timestamps/status only. |
</threat_model>
