<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public internet to Caddy | Anonymous traffic reaches static files and maybe login, but not private originals or retired operator APIs. |
| Caddy to Rust controller | Only `/admin/api/*` reaches the controller on the private Podman network. |
| GitHub Actions to runtime | CI/CD deploys images/config only and cannot read catalog content for publishing. |
| local candidate validation to public cutover | Candidate release must be validated before it becomes public current release. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-19 | Information Disclosure | Caddy static/media routes | mitigate | Serve only generated release tree and generated derivatives; block `/api/operator/*` and retired public catalog APIs after cutover. |
| T-05-20 | Elevation of Privilege | `/admin/api/*` proxy | mitigate | Proxy only to Rust controller and rely on controller auth from Plan 05-02; keep direct container port private. |
| T-05-21 | Tampering | deploy workflow | mitigate | Build/deploy code/images only; do not add GitHub workflow steps that run publisher against Oracle/Object Storage. |
| T-05-22 | Denial of Service | cutover | accept | Planned downtime is acceptable; docs require preflight and roll-forward full rebuild instead of elaborate rollback. |
</threat_model>
