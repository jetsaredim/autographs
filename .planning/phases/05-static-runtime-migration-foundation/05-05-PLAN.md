<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser admin shell to controller | Browser code can call only same-origin `/admin/api/*` and relies on HTTP-only session cookie. |
| admin source to public internet | Static admin shell may be downloadable; no secret values may be embedded. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-16 | Information Disclosure | static admin source | mitigate | Add source tests denying tokens, Object Storage URLs, bucket names, namespaces, and hard-coded passwords. |
| T-05-17 | Cross-Site Request Forgery | admin API | mitigate | Use same-origin requests, `SameSite=Strict` session cookies, and controller-side `Origin`/`Referer` or CSRF-token checks established in Plan 05-02. |
| T-05-18 | Tampering | publish controls | mitigate | Confirm publish/unpublish/rebuild controls call authenticated private endpoints only. |
</threat_model>

    - `admin.js` sends the required CSRF token/header on non-GET `/admin/api/*` calls if the controller exposes one.
