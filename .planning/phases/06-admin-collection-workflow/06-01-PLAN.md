## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin JSON -> controller | Untrusted metadata update JSON crosses into Rust validation and history recording. |
| controller -> Oracle | Trusted controller writes private source-of-truth rows and history events. |
| private history -> admin DTOs | Internal change records become admin-visible JSON later in the phase. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-01-01 | Tampering | `AutographItemUpdate` / `FieldPatch<T>` | high | mitigate | Validate required fields after applying patches; tests must prove null clears optional fields without bypassing title/signer/category validation. |
| T-06-01-02 | Repudiation | `autograph_edit_events` | medium | mitigate | Persist append-only event rows for create/update/publication changes with before/after diffs per D-06-09 and D-06-10. |
| T-06-01-03 | Information Disclosure | history payloads | high | mitigate | Store private object keys only in existing private image rows; do not include object keys, bucket names, namespaces, original filenames, or credentials in `AutographEditEvent` admin-facing fields. |
| T-06-01-04 | Denial of Service | pending-change query | medium | mitigate | Add `autograph_edit_events_item_created_idx` and compute counts with bounded aggregate queries rather than scanning rendered static artifacts. |
| T-06-01-SC | Tampering | package supply chain | low | accept | No package-manager installs are planned; implementation must not add crates or frontend packages in this plan. |
</threat_model>
