<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin login -> session cookie | Password material is exchanged for an HTTP-only admin session. |
| browser session -> management APIs | Session cookie authorizes private collection management mutations. |
| operator environment -> live smoke | Operator-supplied secrets are used only for ignored live verification. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-06-01 | Spoofing | collection-management auth | high | mitigate | Require `AuthKind::Session` for management mutations and reject bearer token management calls. |
| T-06-06-02 | Elevation of Privilege | operator token compatibility | high | mitigate | `AUTOGRAPHS_OPERATOR_API_TOKEN` alone cannot satisfy runtime admin auth and cannot manage items/images/publish. |
| T-06-06-03 | Tampering | CSRF/session mutations | high | mitigate | Preserve SameSite Strict cookies and Origin/Referer checks through `authorize_admin_session`. |
| T-06-06-04 | Information Disclosure | lockout/login errors | medium | mitigate | Keep login failure and lockout responses generic; do not reveal hash, password, token, provider, or internal auth state. |
| T-06-06-SC | Tampering | package supply chain | low | accept | No package-manager installs are planned; use existing auth/controller code. |
</threat_model>
