<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin browser -> status/publish APIs | Browser requests publish and diagnostics data from private controller routes. |
| controller -> static release filesystem | Publisher creates, promotes, and prunes directories under the configured static root. |
| controller -> admin diagnostics DTO | Internal provider/publisher state is redacted before reaching the browser. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-04-01 | Tampering | publish routes | high | mitigate | Preserve authenticated same-origin mutation checks and keep publish separate from save routes. |
| T-06-04-02 | Denial of Service | static release retention | high | mitigate | Add bounded promoted and failed release retention, never deleting `current`, and test pruning behavior. |
| T-06-04-03 | Information Disclosure | `/admin/api/status` | high | mitigate | Dedicated redacted status DTOs and tests deny secrets, Object Storage identifiers, Oracle internals, and object keys. |
| T-06-04-04 | Repudiation | publish batching | medium | mitigate | Record successful publish job state so pending changes and history have a durable publication boundary. |
| T-06-04-SC | Tampering | package supply chain | low | accept | No package-manager installs are planned; use existing Rust/config/publisher modules. |
</threat_model>
