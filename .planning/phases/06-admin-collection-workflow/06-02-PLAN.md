<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin browser -> `/admin/api/items*` | Authenticated but untrusted query parameters and JSON request bodies enter controller routes. |
| controller -> repository | Route DTOs are transformed into repository filters and updates. |
| repository -> admin response | Private source-of-truth records are redacted before browser display. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-02-01 | Spoofing | item/history routes | high | mitigate | Require existing admin authentication on all new `/admin/api/items*` GET and mutation routes; unauthenticated requests return 401. |
| T-06-02-02 | Tampering | item update route | high | mitigate | Use `AutographItemUpdate` validation from Plan 06-01 and preserve same-origin CSRF checks for mutations. |
| T-06-02-03 | Information Disclosure | list/detail/history DTOs | high | mitigate | Use dedicated DTOs that exclude object keys, bucket names, namespaces, original filenames, credentials, and Oracle internals; add response privacy assertions. |
| T-06-02-04 | Denial of Service | item finder filters | medium | mitigate | Keep filters simple and bounded to in-repo fields; avoid public search infrastructure and advanced query syntax per D-06-26. |
| T-06-02-SC | Tampering | package supply chain | low | accept | No package-manager installs are planned; implementation must use existing Axum, Serde, and Rust stack. |
</threat_model>
