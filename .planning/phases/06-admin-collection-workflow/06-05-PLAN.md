<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin API JSON -> DOM | Catalog metadata, history values, and diagnostics values are rendered in the browser. |
| admin browser -> private controller | Browser sends same-origin authenticated requests for saves, image actions, and publish. |
| static admin source -> public repo | Static source is committed and must not embed secrets or private storage identifiers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-05-01 | Elevation of Privilege | admin browser workflow | high | mitigate | Use only same-origin `/admin/api/*` calls with credentials included; no public account, role, token, or external auth UI is added. |
| T-06-05-02 | Tampering | form/image/publish actions | high | mitigate | Browser sends explicit save, image, cleanup retry, and publish requests; no save path triggers publish automatically. |
| T-06-05-03 | Information Disclosure | static admin source and DOM | high | mitigate | Static source tests deny secrets, Object Storage identifiers, browser storage, and privileged non-admin API paths; render untrusted text via `textContent`/node creation. |
| T-06-05-04 | Denial of Service | multi-image upload UI | medium | mitigate | UI loops over selected files and relies on existing per-request body limit and server image validation; no bulk import or background queue is introduced. |
| T-06-05-SC | Tampering | frontend supply chain | low | accept | No frontend package, icon library, registry block, build tool, React, Vite, Tailwind, or shadcn dependency is planned. |
</threat_model>
